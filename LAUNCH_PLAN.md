# ModelSweeper Launch Plan (14 Days)

## Goal
Create sustained visibility, collect qualified feedback, and convert interest into contributors and early adopters.

## Positioning
- Category: Desktop AI workflow operations
- Hook: "One control center for model presets, memory fit, and integration sync"
- Audience:
  - Local AI builders
  - AI engineers and developer tooling users
  - Productivity-focused technical creators

## Core Narrative Pillars
1. Workflow clarity: role-based model orchestration
2. Reliability: memory/hardware feasibility before runtime
3. Speed: one place to configure and synchronize tools

## Launch Assets Checklist
- [x] Landing page (`docs/`)
- [x] Social preview image (`docs/assets/social-preview.png`)
- [x] README + release page
- [ ] 60-second product walkthrough video
- [ ] 3 feature GIFs (presets, hardware calculator, integrations)
- [ ] Screenshot pack for social posts

## 14-Day Execution

### Day 1
- Publish `v0.1.0` and pin release.
- Post launch thread (X + LinkedIn + Reddit relevant communities).
- Share direct CTA: "Try + open feedback issue".

### Day 2
- Short demo clip: preset switching + role assignment.
- Reply to every comment with concrete examples.

### Day 3
- Technical post: how hardware calculator estimates memory.
- Cross-post to dev communities.

### Day 4
- Community ask: "What integration should be next?"
- Collect votes via GitHub Discussions.

### Day 5
- Publish roadmap issue and tag good-first issues.
- Invite first contributors.

### Day 6
- Share “before/after workflow” story from real usage.

### Day 7
- Recap post: week-1 metrics and top requests.

### Day 8
- Ship a visible improvement based on feedback.
- Post “We listened” changelog snippet.

### Day 9
- Publish side-by-side comparison with manual workflows.

### Day 10
- Creator collab: ask 2-3 small creators to test live.

### Day 11
- Dev-focused deep dive: architecture overview.

### Day 12
- Community sprint day: close quick wins + thank contributors publicly.

### Day 13
- Publish second demo with updated UX flow.

### Day 14
- Release `v0.1.1` feedback-driven update.
- Post “from launch to momentum” summary.

## Weekly KPI Targets
- Stars
- Discussions opened
- Issues with reproducible reports
- PRs opened/merged
- Landing page visits (if tracked)
- Demo views/engagement rate
